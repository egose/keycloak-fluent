# Components

Use `ComponentHandle` when you need to manage realm components that are identified by a combination of name and lookup metadata, not just a single stable id.

This is especially useful for user-storage components and other admin objects where ambiguity is common unless you narrow by `parentId`, `providerId`, or `providerType`.
